# OneHandIME 改进计划

## 现有方案

当前实现了**左手镜像映射**：左手按键映射到自身 + 右手对称位（如 q→[q,p], g→[g,h]），通过 Trie + 词频排序消歧。用户只用左手半区键盘即可输入完整英文。

### 现有方案的问题

1. **学习成本高** — 镜像映射要求用户记住 11 对按键对应关系。盲打的肌肉记忆是"字母→手指动作"的直连通路，左手没有右手字母的运动记忆，镜像映射本质上是从零学习一套新映射。
2. **首选词命中率不够** — 排序完全依赖静态词频，不考虑上下文和用户习惯。短词歧义多时体验差。
3. **选错词无法撤回** — 空格/标点自动提交首选候选后，没有快速回退机制，只能手动退格删词重打。
4. **不自动追加空格** — 每选一个词后要额外按空格，英文场景下等于每词多一次击键。
5. **preedit 显示不直观** — 显示按键序列（如 T-G-E）而非首选候选词（如 the），用户无法快速确认系统是否理解正确。
6. **前缀模式缺少剪枝** — `collectBelow` 收集子树全部词条，全键盘前缀匹配模式下会产生上万候选，性能和排序都有问题。
7. **非词典输入体验差** — 没有候选命中时按空格无反应，用户被困住。逃生通道（反引号切 letterMode）不可发现。

## 改进方案

### 一、新增输入模式

**全键盘前缀预测（面向临时单手用户）**

- 用一只手在全键盘上正常打字，够远的键慢一点而已
- 输入 2~3 个字母后弹出候选词，按数字键/空格选词
- 零学习成本，所有人都会用
- KeyMap 改为每个键只映射自己，复用现有 Trie 的前缀展开能力

**右手镜像布局**

- 现有 `KeyMap.leftHand()` 的镜像版本，供右手用户使用

### 二、提高首选词命中率

按投入产出比排序：

1. **最近词加权** — 维护最近 30~50 个已提交词的环形缓冲区，排序时给这些词加 boost 分。人类写作有极强的局部性，这一点改动极小但效果明显。
2. **用户选词频率学习** — 记录用户每次选词 `word → count++`，存本地文件，排序公式加入用户频率项。
3. **精确长度优先（前缀模式）** — 给长度接近输入长度的候选加分，因为用户往往在接近打完时才看候选。
4. **唯一候选提示（镜像模式）** — 当只剩一个候选时，preedit 直接显示该词而非按键序列。
5. **Bigram 上下文（v2）** — 预计算 `(prevWord, nextWord) → frequency` 表，排序时结合上一个已提交词的上下文。研究表明可减少约 30% 的选词错误。需要额外数据，作为后续版本重点。

### 三、体验细节

1. **自动追加空格** — 选词提交后自动加空格。
2. **选词撤回** — 提交后支持快速回退到上一次候选状态。
3. **内联 preedit** — 消歧模式下 preedit 显示首选候选词，而非按键序列。
4. **无候选时提交原始输入** — 没有候选命中时，按空格/回车提交原始键入字母，而非无反应。
5. **前缀搜索剪枝** — `collectBelow` 加结果数量上限（v1），或 Trie 节点预存 top-K（v2）。

### 四、产品定位

- **目标用户**：偏瘫、手部损伤、中风康复等长期/永久单手使用者，以及临时单手场景用户
- **核心价值**：省心——开箱就能用，不给用户添更多麻烦
- **三种模式覆盖所有场景**：
  - 左手镜像：适合愿意学习、追求效率的永久用户
  - 右手镜像：同上，右手版本
  - 全键盘前缀预测：适合临时用户，零学习成本
- **非词典输入**：引导用户切换到系统标准输入法，macOS 输入法切换快捷键即可
- **定价参考**：$29~49 一次性买断，macOS 直接分发（官网 + DMG）
